Limitations & Honest Challenges with Opus 4.7

1. Cost at Scale

While Opus 4.7 is efficient, running agents continuously gets expensive. A single agent processing documents 24/7 costs ~$5,000/month without optimization. Layers of agents scale that cost quickly.

Solution: Implement the tiering strategy above — use Sonnet by default, Opus 4.7 selectively.

2. Token Counting & Billing Surprises

Opus 4.7 uses tokens liberally (by design, for reasoning). If you're not tracking token usage, your bill can spiral quickly.

Solution: Implement token counting on every request. Use structured outputs to reduce verbosity.

3. Latency on Complex Tasks

Adaptive thinking means latency varies. Simple tasks are fast; complex ones can take 20-30 seconds. For real-time customer-facing applications, this is a challenge.

Solution: Use Opus 4.7 for async/background work. Use Sonnet 4.6 for real-time interactions.

4. Training Data Cutoff (Jan 2026)

Opus 4.7's knowledge cuts off at January 2026. For current events, real-time data, or recent releases, it needs tool access (web search, APIs, databases).
